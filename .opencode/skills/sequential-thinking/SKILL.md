# Sequential Thinking

## What It Does
Provides structured thinking process for complex problems - think, revise, branch, and verify.

## When to Use
- Complex debugging
- Multi-step planning
- Architecture decisions
- Logic problems
- Code review decisions

## How to Use
The MCP provides a `sequential_thinking` tool that:
1. Breaks down problems into steps
2. Can revise earlier thoughts
3. Can branch into alternatives
4. Can verify before concluding

## Example
```
Use sequential thinking to debug why the upload is failing
```

## When NOT to Use
- Simple lookups
- Single-step tasks
- Quick questions

## Best Practices
- Use for problems with multiple possible solutions
- Let it think through edge cases
- Combine with domain-specific MCPs
