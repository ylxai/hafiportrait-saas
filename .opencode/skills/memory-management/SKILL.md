# Memory Management

## What It Does
Provides persistent context across conversations - remembers important information.

## When to Use
- Remembering project conventions
- Storing team preferences
- Maintaining context across sessions
- Cross-agent knowledge sharing

## How to Use
The memory MCP provides:
- Store important facts
- Query stored memories
- Update existing memories
- Delete old entries

## Best Practices
1. Store only important, persistent info
2. Use clear, searchable keys
3. Review memories periodically
4. Don't store sensitive data

## Example
```
Remember that we use @/ alias for all src/ imports
Store this as a project convention
```
