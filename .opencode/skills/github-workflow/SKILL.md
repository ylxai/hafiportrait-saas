# GitHub Workflow

## What It Does
Manage GitHub issues, pull requests, and repository operations.

## When to Use
- Creating issues
- Creating/reviewing PRs
- Checking repository status
- Managing branches

## Available Actions
- Create issue
- Create PR
- Review PR
- List issues
- Check branch status

## How to Use
Add "use github" to your prompt:

```
Create an issue for the login bug. use github
```

```
Create a PR for the new feature. use github
```

## Best Practices
- Use descriptive titles
- Include steps to reproduce (for bugs)
- Link related issues
- Request review from relevant team member
